# 08_COORDINATE_SNAPPING_VALIDATION.md

## 단계명

사용자 좌표 스냅핑 검증

## 목적

사용자의 GPS 좌표를 실제 보행 가능한 시작/도착 위치에 붙일 수 있는지 확인한다.

## 구현 접근

1. 도로 위가 아닌 일반 GPS 좌표 샘플을 준비한다.
2. 출발점과 도착점 각각에 대해 최근접 보행 가능 위치를 찾는다.
3. 허용 가능한 오차 범위와 실패 기준을 정리한다.
4. 성공 케이스와 실패 케이스를 분리해 기록한다.

## 검증할 것

- 임의의 시작점이 도로에서 조금 벗어나 있어도 붙는가
- 목적지가 건물 내부 좌표처럼 찍혀도 붙는가
- 너무 먼 좌표면 실패를 적절히 주는가
- 출발/도착 anchor를 보행 가능한 위치로 바꿀 수 있는가

## 산출물

- 스냅핑 대상 좌표 샘플 목록
- 성공/실패 케이스 기록
- 허용 오차 기준 메모
- 예외 처리 방향 문서

## 완료 기준

- 일반적인 GPS 오차 범위에서 경로 탐색 성공
- 실패 케이스에서 적절한 예외 처리 방향 도출

## 범위 제한

- 실시간 위치 보정이나 UX 설계는 포함하지 않는다.
- 고정밀 보정 알고리즘 최적화는 하지 않는다.

## 다음 단계 입력값

- API 응답으로 넘길 시작/도착 anchor 기준
- snapped point 검토 기준
