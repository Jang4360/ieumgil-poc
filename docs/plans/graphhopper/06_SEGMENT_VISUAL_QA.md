# 06_SEGMENT_VISUAL_QA.md

## 단계명

샘플 세그먼트 지도 시각 검증

## 목적

DB에 적재된 `road_segments.geom`을 지도 위에 선으로 그려서 세그먼트 품질을 시각적으로 검증한다.

## 구현 접근

1. 샘플 구역의 `road_segments.geom`을 조회한다.
2. 지도에서 선 형태로 렌더링한다.
3. 연결 구조, 누락, 길이 이상 여부를 눈으로 점검한다.
4. 이상 케이스를 규칙 또는 생성 로직 후보와 연결해 기록한다.

## 확인할 것

- 길이 계산이 비정상인 세그먼트가 있는지
- 연결이 이상한 구간이 있는지
- 세그먼트가 너무 길거나 짧은지
- 누락 구간이 있는지
- 교차점 분할이 과도하거나 부족한지

## 산출물

- 시각 검증용 지도 캡처 또는 렌더링 결과
- 이상 케이스 목록
- 수정 필요 규칙 메모

## 완료 기준

- 샘플 구역 세그먼트 시각 확인 완료
- 이상 케이스 최소 1회 이상 검토
- 규칙 수정 필요 여부 문서화

## 범위 제한

- 이 단계에서는 GraphHopper 연동까지 확장하지 않는다.
- 프론트엔드 UI 구현은 목표가 아니다.
