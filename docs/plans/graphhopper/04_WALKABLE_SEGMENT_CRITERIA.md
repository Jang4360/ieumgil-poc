# 04_WALKABLE_SEGMENT_CRITERIA.md

## 단계명

샘플 지역 추출 및 보행 가능 세그먼트 기준 검증

## 목적

부산 전체가 아니라 샘플 구역을 먼저 잘라서, 어떤 OSM way를 보행 가능한 세그먼트로 볼지 기준을 정한다.

## 구현 접근

1. 부산 OSM 원본에서 샘플 지역 하나를 먼저 정한다.
2. 샘플 지역만 별도로 추출해 반복 검증에 사용한다.
3. 샘플 데이터에서 보행 관련 way와 교차점을 확인한다.
4. `포함`, `제외`, `예외 검토` 기준표를 만든다.

예시 샘플 지역:
- 서면역 주변 1km ~ 2km

## 확인할 것

- 샘플 구역 추출 가능 여부
- 어떤 태그를 보행 가능으로 볼지
- 어떤 태그를 제외할지
- 계단, 횡단보도, 보행자도로 처리 기준
- `highway`, `foot`, `sidewalk`, `access`, `crossing`, `steps`, `elevator` 조합 처리 방식

## 산출물

- 샘플 구역 파일 또는 샘플 구역 정의 문서
- 보행 가능 태그 후보 목록
- 제외 태그 후보 목록
- 예외 처리 대상 태그 목록
- 1차 추출 규칙 문서

## 완료 기준

- 샘플 구역 1개 이상 확정
- 보행 가능/불가 태그 기준 문서화
- 계단/횡단보도/보행자도로 처리 원칙 정리
- 1차 추출 규칙 확정

## 범위 제한

- 아직 `road_nodes`, `road_segments` 생성 구현으로 확장하지 않는다.
- GraphHopper profile 최적화는 이 단계에 포함하지 않는다.
