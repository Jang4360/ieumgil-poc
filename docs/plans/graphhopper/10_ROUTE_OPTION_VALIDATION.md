# 10_ROUTE_OPTION_VALIDATION.md

## 단계명

`SAFE` / `SHORTEST` 분기 가능성 검증

## 목적

단순 최단 경로 외에 안전 우선 경로를 구조적으로 분리할 수 있는지 확인한다.

## 구현 접근

1. 기본 보행 경로를 `SHORTEST` 기준으로 둔다.
2. `SAFE`는 계단 회피 또는 일부 도로 회피 룰을 가정하고 비교한다.
3. profile 분리, custom model, 후처리 회피 규칙 중 어떤 방식이 가능한지 검토한다.
4. 실제 결과 차이가 작더라도 구조 분리 가능성 자체를 평가한다.

## 확인할 것

- 서로 다른 profile 또는 custom model로 두 결과를 만들 수 있는가
- 지금은 둘 다 비슷하게 나와도 구조적으로 분리 가능한가
- 계단 회피 같은 제약을 일부라도 적용할 수 있는가
- 위험 구간 정보는 세그먼트 후처리로 붙일 수 있는가

## 산출물

- `SAFE` / `SHORTEST` 분리 전략 메모
- 적용 가능한 제약 후보 목록
- 추가 데이터 또는 후처리 필요 항목 기록

## 완료 기준

- `SAFE` / `SHORTEST` 분기 구조 가능 여부 판단
- 최소 1개 이상 회피 규칙 후보 정리
- 후처리 필요 지점 문서화

## 범위 제한

- 경사도, 점자블록, 공공데이터까지 한 번에 모두 반영하지 않는다.
- 운영 수준의 추천 품질 비교는 목표가 아니다.
