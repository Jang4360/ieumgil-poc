# 02_TRANSIT_ROUTE_CANDIDATE_VALIDATION.md

## 단계명

버스 / 지하철 경로 후보 조회 가능성 검증

## 목적

출발지 / 도착지 기준으로 버스, 지하철, 환승 조합이 포함된 대중교통 후보 경로를 실제로 가져올 수 있는지 확인한다.

## 구현 접근

1. Step 1에서 선정한 우선 후보 원천으로 샘플 출발지 / 도착지를 준비한다.
2. 최소 2~3개 시나리오에 대해 대중교통 후보 경로 조회를 시도한다.
3. 경로 후보 수, 환승 횟수, 소요시간, 정류장 / 역 / 노선 포함 여부를 확인한다.
4. 응답 내 구간 단위 타입 분리 가능 여부를 기록한다.
5. 응답에서 바로 쓸 수 있는 값과 후처리 보강이 필요한 값을 구분한다.

## 확인할 것

- 좌표 또는 주소 기반으로 경로 후보 조회가 가능한지
- 버스 / 지하철 / 환승 구간이 응답에서 구분되는지
- 정류장 / 역 / 노선명 또는 ID가 포함되는지
- 도보 구간 정보가 별도 타입 또는 섹션으로 제공되는지
- 경로 후보 수와 기본 메타 정보가 MVP 판단에 충분한지

## 샘플 시나리오

- 부산역 → 서면역
- 서면역 → 해운대역
- 버스 환승이 포함될 가능성이 높은 도심 구간 1개

## 산출물

- 샘플 조회 요청 / 응답 기록
- 후보 경로 수와 핵심 필드 확인 결과
- 응답 구조 요약 문서
- 실패 케이스 또는 조회 불가 케이스 기록

## 완료 기준

- 최소 1개 이상의 대중교통 후보 경로 조회 성공
- 버스 / 지하철 / 환승 정보 포함 여부 문서화
- 도보 구간 추출 가능성 판단을 위한 응답 구조 정리

## 범위 제한

- 요금 비교, 혼잡도, 실시간 도착 예측은 현재 단계에 포함하지 않는다.
- 추천 품질 자체를 평가하지 않고 조회 가능 여부와 구조 확인에 집중한다.

## 다음 단계 입력값

- 정류장 / 역사 / 노선 관련 응답 필드 목록
- 데이터셋 간 연결 전략 검토에 사용할 실응답 샘플
