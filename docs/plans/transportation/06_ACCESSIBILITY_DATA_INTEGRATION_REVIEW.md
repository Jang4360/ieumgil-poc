# 06_ACCESSIBILITY_DATA_INTEGRATION_REVIEW.md

## 단계명

접근성 정보 반영 가능성 검토

## 목적

저상버스, 역사 엘리베이터, 지하철 역사 접근성, 정류장 / 역 출입구 정보 중 현재 단계에서 어디까지 반영할 수 있는지 구분한다.

## 구현 접근

1. Step 1과 Step 3에서 확인한 접근성 데이터 원천을 다시 정리한다.
2. 버스, 지하철, 도보 각 구간에 붙일 수 있는 접근성 속성을 분리한다.
3. 지금 바로 반영 가능한 값과 후속 확장 값으로 나눈다.
4. 응답 필드 수준에서 단순 플래그, 상세 메타 정보, 별도 보정 테이블 중 어떤 형태가 적절한지 검토한다.
5. 데이터 부재나 품질 불일치 사례를 따로 남긴다.

## 확인할 것

- 저상버스 여부를 노선 또는 운행 단위로 확인할 수 있는지
- 역사 엘리베이터 / 출입구 접근성 정보를 역 단위 또는 출입구 단위로 얻을 수 있는지
- 버스 / 지하철 / 도보 각 구간에 붙일 접근성 속성이 무엇인지
- 현재 응답에서 단순 노출 가능한 값과 후속 보강이 필요한 값이 무엇인지
- 데이터가 없어도 구조만 먼저 정의할 수 있는 필드가 무엇인지

## 산출물

- 접근성 속성 후보 목록
- 즉시 반영 가능 값 / 후속 확장 값 구분표
- 데이터 부재 및 품질 이슈 기록
- 구간별 접근성 부착 방향 메모

## 완료 기준

- 접근성 정보 반영 범위의 1차 구분 완료
- 구간별로 어떤 접근성 값을 붙일지 방향 정리
- 후속 확장이 필요한 요소 문서화

## 범위 제한

- 실시간 저상버스 배차 반영은 하지 않는다.
- 역사 내부 길찾기 수준의 상세 접근성 모델은 포함하지 않는다.
