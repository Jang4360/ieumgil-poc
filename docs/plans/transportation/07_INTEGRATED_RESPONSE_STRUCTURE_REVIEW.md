# 07_INTEGRATED_RESPONSE_STRUCTURE_REVIEW.md

## 단계명

대중교통 + 도보 통합 응답 구조 방향 검토

## 목적

버스, 지하철, 도보 구간을 하나의 응답 구조로 표현할 수 있는지 확인하고 MVP 범위를 정리한다.

## 구현 접근

1. Step 2, Step 4, Step 5, Step 6의 결과를 구간 단위로 모은다.
2. `WALK`, `BUS`, `SUBWAY` 같은 구간 타입 분리 방식을 검토한다.
3. 모든 구간에 공통으로 필요한 필드와 구간별 전용 필드를 나눈다.
4. 접근성 정보와 환승 정보를 어느 레벨에 둘지 정리한다.
5. MVP에서 반드시 필요한 응답과 후속 확장 응답을 구분한다.

## 확인할 것

- 구간 타입을 일관되게 분리할 수 있는지
- 도보, 버스, 지하철 구간에 공통 필드가 무엇인지
- 환승 정보와 접근성 정보를 어느 레벨에 배치할지
- 후보 경로 재정렬을 위해 필요한 최소 메타 정보가 무엇인지
- 후속 확장 필드를 지금 구조에 무리 없이 추가할 수 있는지

## 산출물

- 통합 응답 구조 초안
- 구간 타입별 필드 정의 메모
- MVP 필수 필드 / 후속 확장 필드 구분표
- 응답 구조 한계와 보완 필요 사항 기록

## 완료 기준

- 대중교통 + 도보 통합 응답 구조 초안 작성
- 구간 타입 분리 방식과 필수 필드 정의 정리
- MVP 범위와 후속 확장 범위 문서화

## 범위 제한

- 메인 서비스의 최종 API 명세 확정은 하지 않는다.
- 점수화나 추천 재정렬 알고리즘은 구조 검토 수준까지만 다룬다.
