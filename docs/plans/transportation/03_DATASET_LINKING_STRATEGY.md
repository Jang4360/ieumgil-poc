# 03_DATASET_LINKING_STRATEGY.md

## 단계명

ODsay / BIMS / 지하철 접근성 데이터 연결 전략 검토

## 목적

서로 다른 데이터셋에 있는 정류장, 역사, 노선, 접근성 정보를 현실적으로 연결할 수 있는지 판단한다.

## 구현 접근

1. Step 2의 실제 경로 응답에서 사용하는 정류장 / 역사 / 노선 필드를 추린다.
2. BIMS와 지하철 공개 데이터의 식별자 체계를 나란히 비교한다.
3. ID 직접 매칭, 명칭 매칭, 좌표 매칭, 보조 키 조합 전략을 각각 검토한다.
4. 완전 자동 매핑이 어려운 지점을 실패 케이스로 분리한다.
5. MVP에서 쓸 수 있는 최소 연결 전략과 후속 보정 전략을 구분한다.

## 확인할 것

- 정류장 ID, 역 ID, 노선 ID 또는 노선명 직접 매칭 가능 여부
- 명칭 표기 차이와 동일 명칭 다건 존재 여부
- 좌표 기반 최근접 매칭이 현실적인지
- 운영기관별 코드 체계 차이가 큰지
- 수동 매핑 테이블이 필요한 구간이 어디인지

## 산출물

- 연결 키 후보 비교표
- 매칭 전략 우선순위 문서
- 자동 매핑 실패 유형 목록
- 수동 보정 필요 후보 목록

## 완료 기준

- 데이터셋 간 연결 전략 1개 이상 도출
- 좌표 기반 연결과 보조 키 연결의 역할 분담 정리
- 자동 매핑 한계와 수동 보정 필요 지점 문서화

## 범위 제한

- 전체 데이터셋의 완전한 통합 테이블 구축은 현재 단계에 포함하지 않는다.
- 운영 수준의 정합성 보장은 목표가 아니다.

## 다음 단계 입력값

- 도보 구간 anchor 추출 시 사용할 정류장 / 역사 연결 기준
- 접근성 정보 결합 가능 여부 판단용 매핑 전략
