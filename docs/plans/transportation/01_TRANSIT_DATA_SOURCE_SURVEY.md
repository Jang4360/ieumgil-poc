# 01_TRANSIT_DATA_SOURCE_SURVEY.md

## 단계명

대중교통 조회 원천 1차 확정 및 접근 확인

## 목적

ODsay, 부산버스정보시스템(BIMS), 부산교통공사_부산도시철도 운행 정보를 대중교통 PoC의 1차 입력 원천으로 사용하기 위해
각 API의 역할, 인증 방식, 조회 가능 범위를 정리한다.

## 구현 접근

1. ODsay를 출발지 / 도착지 기반 대중교통 경로 조회와 전체 이동시간 조회의 기준 원천으로 둔다.
2. 부산 BIMS를 버스 정류장 조회, 버스 노선 조회, 실시간 버스 도착시간 조회 원천으로 둔다.
3. 부산교통공사_부산도시철도 운행 정보를 정거장 도착시각, 정거장 출발시각, 운행구간 정거장 목록, 운행속도 조회 원천으로 둔다.
4. 각 API의 인증 방식, 엔드포인트, 응답 포맷, 핵심 연결 키를 정리한다.
5. PoC에서 어떤 조회를 어느 API로 처리할지 역할 분담을 문서화한다.

## 확인할 것

- ODsay에서 출발지 / 도착지 조회와 대중교통 경로 조회를 할 수 있는지
- ODsay에서 전체 이동시간과 구간별 이동시간을 받을 수 있는지
- 부산 BIMS에서 버스 정류장 조회, 노선 조회, 실시간 도착시간 조회를 할 수 있는지
- 부산교통공사_부산도시철도 운행 정보에서 정거장 도착시각, 정거장 출발시각, 운행구간 정거장 목록, 운행속도를 조회할 수 있는지
- 각 API에 정류장 ID, 역 ID, 노선명, 좌표, 경로 구간 시간 같은 연결 키가 있는지
- 세 API의 응답 포맷이 PoC 검증에 바로 사용할 수준인지

## 산출물

- 확정된 1차 API 원천 목록
- API별 제공 범위 요약표
- 인증 / 호출 방식 / 포맷 정리 문서
- 조회 유형별 API 역할 분담표

## 완료 기준

- 출발지 / 도착지 기반 경로 조회용 API 1개 이상 확정
- 버스 실시간 도착시간 조회용 API 1개 이상 확정
- 지하철 운행 시각 / 정거장 도착시각 조회용 API 1개 이상 확정
- 이동시간 조회에 사용할 API와 필드 후보를 문서화

## 범위 제한

- 아직 실제 통합 응답 DTO 매핑은 세부 구현하지 않는다.
- 실시간 정보의 정확도, 지연 반영 품질, 운영 단계 SLA 평가는 현재 범위에 포함하지 않는다.

## 다음 단계 입력값

- 출발지 / 도착지 조회와 경로 조회에 사용할 API 목록
- 버스 / 지하철 도착시간 조회에 사용할 API 목록
- 이동시간 조회에 사용할 핵심 필드 목록
