# 01_TRANSIT_DATA_SOURCE_SURVEY.md

## 단계명

대중교통 데이터 원천 후보 조사

## 목적

ODsay, BIMS, 지하철 관련 공개 데이터가 대중교통 PoC의 실제 입력 원천이 될 수 있는지 확인한다.

## 구현 접근

1. 대중교통 경로 조회용 원천과 접근성 보강용 원천을 먼저 분리해서 본다.
2. ODsay, BIMS, 지하철 공개 데이터 각각의 제공 범위와 포맷을 정리한다.
3. 정류장, 역사, 노선, 출입구, 접근성 정보가 어떤 수준까지 제공되는지 기록한다.
4. 호출 제약, 인증 방식, 좌표 제공 여부, 갱신 주기를 함께 정리한다.
5. PoC에서 바로 사용할 후보와 후속 검토로 넘길 후보를 나눈다.

## 확인할 것

- ODsay에서 출발지 / 도착지 기반 대중교통 경로 후보를 조회할 수 있는지
- BIMS에서 버스 노선, 정류장, 운영 정보를 얻을 수 있는지
- 지하철 역사, 출입구, 엘리베이터, 접근성 관련 공개 데이터가 존재하는지
- 각 데이터셋에 정류장 ID, 역 ID, 노선명, 좌표 같은 연결 후보 키가 있는지
- 응답 포맷이 PoC 검증에 바로 사용할 수준인지

## 산출물

- 데이터 원천 후보 목록
- 데이터 원천별 제공 범위 요약표
- 인증 / 호출 방식 / 포맷 정리 문서
- PoC 우선 사용 후보와 후속 검토 후보 구분표

## 완료 기준

- 버스 / 지하철 경로 조회용 데이터 원천 1개 이상 식별
- 접근성 보강용 데이터 원천 1개 이상 식별
- 각 데이터셋의 핵심 연결 키 후보를 문서화

## 범위 제한

- 아직 실제 대중교통 경로 응답 구조를 세부 매핑하지 않는다.
- 실시간 정보의 정확도나 운영 품질 평가는 현재 범위에 포함하지 않는다.

## 다음 단계 입력값

- 후보 경로 조회에 사용할 API 또는 데이터셋
- 정류장 / 역사 / 노선 관련 핵심 필드 목록
